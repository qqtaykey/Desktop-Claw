# 文件操作技能（FileSkill）

## 概述

此技能让你能够读取、创建和编辑用户本地文件系统中的文件。所有文件操作都限制在用户授权的目录范围内。

## 可用工具

- `read_file` — 读取指定路径的文件内容。纯文本文件直接返回内容；.pdf / .docx / .xlsx 返回提取的文本。
- `write_file` — 创建新文件或覆写已有文件。适用于用户要求「帮我写一份 XX」「创建一个 XX」的场景。
- `edit_file` — 通过字符串替换精确修改已有文件。适用于「帮我改一下 XX」「把 A 替换成 B」的场景。

## 使用指南

### 何时使用 read_file

- 用户说「打开」「看看」「读一下」某个文件
- 用户拖拽文件到对话（消息中包含文件路径）
- 你需要了解某个文件的内容才能回答问题

### 何时使用 write_file

- 用户明确要求创建新文件（「帮我写一份笔记」「创建一个 TODO 列表」）
- 用户要求将内容保存到文件
- **注意**：覆写已有文件前，先用 read_file 确认文件是否存在。如果文件存在且用户未明确说要覆写，应先告知用户

### 何时使用 edit_file

- 用户要求修改已有文件的部分内容
- 先用 read_file 读取文件，找到要修改的精确文本片段，再用 edit_file 替换
- old_text 必须与文件中的内容完全匹配（包括空格和换行）

## 文件类型支持

| 类型 | 读取 | 写入 | 编辑 |
|------|------|------|------|
| .txt, .md | ✅ 直接读取 | ✅ | ✅ |
| 代码文件 (.js, .ts, .py 等) | ✅ 直接读取 | ✅ | ✅ |
| .pdf | ✅ 文本提取 | ❌ | ❌ |
| .docx | ✅ 文本提取 | ❌ | ❌ |
| .xlsx | ✅ 文本提取 | ❌ | ❌ |

## 安全边界

- 所有路径必须在用户授权的目录（allowedRoots）范围内，路径校验失败会返回错误
- 禁止访问系统敏感目录（如 ~/.ssh/、/etc/）
- 不要尝试读取二进制文件（图片、视频、可执行文件等）
- 文件大小有上限限制，过大的文件会返回截断提示
